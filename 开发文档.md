## 🏗️ 项目架构

### 技术栈

- **语言**: Python 3.10+
- **数据库**: PostgreSQL (Supabase)
- **AI**: DeepSeek API
- **自动化**: GitHub Actions
- **通知**: SMTP + 飞书 Webhook

### 目录结构

```
ai-email-coach/
├── .github/workflows/      # GitHub Actions 工作流
│   ├── daily_review.yml    # 每日复盘（21:30）
│   ├── daily_followup.yml  # 每日跟进（23:00）
│   ├── check_reply.yml     # 检查回复（23:30）
│   ├── weekly_report.yml   # 周报（周日20:00）
│   └── monthly_report.yml  # 月报（月末20:00）
├── scripts/                # Python 脚本
│   ├── daily_review.py     # 每日复盘
│   ├── daily_followup.py   # 每日跟进
│   ├── check_email_reply.py # 邮件解析
│   ├── weekly_report.py    # 周报生成
│   ├── monthly_report.py   # 月报生成
│   └── gamification_utils.py # 游戏化工具函数
├── database_setup.sql      # 数据库初始化
├── requirements.txt        # Python 依赖
├── .env.example           # 环境变量示例
├── 部署指南.md            # 部署文档
├── 用户手册.md            # 用户手册
└── README.md              # 项目说明
```

## 💾 数据库设计

### 核心表结构

#### tasks（任务表）
```sql
- id: UUID (主键)
- user_email: TEXT (用户邮箱)
- task_name: TEXT (任务名称)
- quadrant: INTEGER (象限 1-4)
- task_order: INTEGER (任务编号)
- display_number: TEXT (显示编号，如 Q1-1)
- progress_percentage: INTEGER (进度 0-100)
- status: TEXT (状态: active/paused/completed)
- is_deleted: BOOLEAN (软删除标记)
- created_at: TIMESTAMP
- updated_at: TIMESTAMP
```

#### user_gamification（游戏化数据）
```sql
- id: UUID (主键)
- user_email: TEXT (用户邮箱)
- level: INTEGER (等级)
- current_exp: INTEGER (当前经验值)
- total_exp: INTEGER (总经验值)
- coins: INTEGER (金币)
- ai_personality: TEXT (AI性格)
- consecutive_q1_days: INTEGER (Q1连击天数)
- created_at: TIMESTAMP
- updated_at: TIMESTAMP
```

#### user_reply_tracking（回复追踪）
```sql
- id: UUID (主键)
- user_email: TEXT (用户邮箱)
- last_reply_date: DATE (最后回复日期)
- consecutive_no_reply_days: INTEGER (连续未回复天数)
- total_replies: INTEGER (总回复次数)
```

#### shop_items（商店道具）
```sql
- id: UUID (主键)
- item_code: TEXT (道具代码)
- item_name: TEXT (道具名称)
- price: INTEGER (价格)
- required_level: INTEGER (所需等级)
- usage_limit_type: TEXT (限制类型)
- usage_limit_count: INTEGER (限制次数)
```

#### user_inventory（用户背包）
```sql
- id: UUID (主键)
- user_email: TEXT (用户邮箱)
- item_code: TEXT (道具代码)
- quantity: INTEGER (数量)
```

### 数据隔离

所有表都使用 `user_email` 作为用户标识，确保：
- 每个用户只能访问自己的数据
- 使用 Supabase RLS 策略进行访问控制
- service_role key 拥有完全访问权限

添加 GitHub Actions workflow：

```yaml
name: Custom Report
on:
  schedule:
    - cron: '0 10 * * *'  # 自定义时间
jobs:
  run:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Set up Python
        uses: actions/setup-python@v4
        with:
          python-version: '3.10'
      - name: Install dependencies
        run: pip install -r requirements.txt
      - name: Run custom report
        env:
          SUPABASE_URL: ${{ secrets.SUPABASE_URL }}
          SUPABASE_KEY: ${{ secrets.SUPABASE_KEY }}
          EMAIL_163_USERNAME: ${{ secrets.EMAIL_163_USERNAME }}
          EMAIL_163_PASSWORD: ${{ secrets.EMAIL_163_PASSWORD }}
        run: python scripts/custom_report.py
```
