## 🎯 前置准备

### 1. 注册必要的服务

你需要注册以下服务（全部免费）：

- **GitHub账号** - 用于托管代码和运行自动化
- **Supabase账号** - 用于存储任务数据（免费额度足够个人使用）
- **DeepSeek账号** - 用于AI解析邮件（新用户有免费额度）
- **163邮箱** - 用于接收和发送邮件
- **飞书机器人**（可选）- 用于接收通知

### 2. 获取API密钥

#### Supabase
1. 访问 https://supabase.com 并登录
2. 创建新项目（选择离你最近的区域）
3. 进入项目设置 → API
4. 复制以下信息：
   - `Project URL` → 这是你的 `SUPABASE_URL`
   - `service_role key` → 这是你的 `SUPABASE_KEY`（⚠️ 不要泄露）

#### DeepSeek
1. 访问 https://platform.deepseek.com 并登录
2. 进入 API Keys 页面
3. 创建新的 API Key → 这是你的 `DEEPSEEK_API_KEY`

#### 163邮箱授权码
1. 登录 163 邮箱网页版
2. 设置 → POP3/SMTP/IMAP
3. 开启 SMTP 服务
4. 获取授权码（不是邮箱密码）→ 这是你的 `EMAIL_163_PASSWORD`

#### 飞书机器人（可选）
1. 在飞书群中添加自定义机器人
2. 获取 Webhook URL → 这是你的 `FEISHU_WEBHOOK_URL`

## 💾 数据库配置

### 1. Fork 项目到你的 GitHub

1. 访问项目地址：https://github.com/Zihui1112/ai-email-coach
2. 点击右上角的 `Fork` 按钮
3. 等待 Fork 完成

### 2. 初始化数据库

1. 打开 Supabase 项目控制台
2. 进入 SQL Editor
3. 复制项目中的 `database_setup.sql` 文件内容
4. 粘贴到 SQL Editor 并执行
5. 等待执行完成（应该看到所有表都创建成功）

### 3. 配置行级安全策略（RLS）

Supabase 默认启用了行级安全策略，需要配置以允许 service_role 访问：

1. 在 SQL Editor 中执行 `fix_rls_policy.sql` 文件内容
2. 这会为所有表添加正确的访问策略

## ⚙️ GitHub配置

### 1. 配置 GitHub Secrets

在你 Fork 的仓库中：

1. 进入 `Settings` → `Secrets and variables` → `Actions`
2. 点击 `New repository secret`
3. 添加以下 Secrets：

| Secret名称 | 说明 | 示例 |
|-----------|------|------|
| `SUPABASE_URL` | Supabase项目URL | https://xxx.supabase.co |
| `SUPABASE_KEY` | Supabase service_role key | eyJhbGc... |
| `DEEPSEEK_API_KEY` | DeepSeek API密钥 | sk-xxx |
| `EMAIL_163_USERNAME` | 163邮箱地址 | your@163.com |
| `EMAIL_163_PASSWORD` | 163邮箱授权码 | ABCD1234 |
| `FEISHU_WEBHOOK_URL` | 飞书Webhook（可选） | https://open.feishu.cn/... |

⚠️ **重要提示**：
- 每个用户使用自己的 Secrets，数据完全隔离
- 不要在代码中硬编码任何密钥
- service_role key 拥有完全访问权限，务必保密

### 2. 启用 GitHub Actions

1. 进入仓库的 `Actions` 标签
2. 如果看到提示，点击 `I understand my workflows, go ahead and enable them`
3. GitHub Actions 会自动按照设定的时间运行

### 3. 调整时区（可选）

GitHub Actions 默认使用 UTC 时间，如果需要调整：

1. 编辑 `.github/workflows/` 下的 workflow 文件
2. 修改 `cron` 表达式（当前已设置为北京时间）

## 🔒 数据隔离说明

### 多用户部署

每个用户 Fork 项目后：

1. **独立的数据库**：每个用户使用自己的 Supabase 项目
2. **独立的配置**：每个用户配置自己的 GitHub Secrets
3. **独立的邮箱**：每个用户使用自己的邮箱接收通知
4. **完全隔离**：用户之间的数据完全独立，互不影响

### 数据安全

- 所有敏感信息存储在 GitHub Secrets 中
- Supabase 使用行级安全策略（RLS）
- 数据库访问需要 service_role key
- 邮件和飞书通知只发送给配置的用户

## ❓ 常见问题

### Q1: GitHub Actions 没有自动运行？

**A**: 检查以下几点：
1. 确认 Actions 已启用
2. 检查 workflow 文件中的 cron 表达式
3. Fork 的仓库需要至少有一次 commit 才会触发定时任务
4. 可以手动触发测试

### Q2: 收不到邮件？

**A**: 检查以下几点：
1. 163邮箱授权码是否正确
2. 检查垃圾邮件文件夹
3. 查看 GitHub Actions 日志是否有错误
4. 确认 SMTP 服务已开启

### Q3: 数据库连接失败？

**A**: 检查以下几点：
1. SUPABASE_URL 和 SUPABASE_KEY 是否正确
2. 使用的是 service_role key（不是 anon key）
3. Supabase 项目是否正常运行
4. 检查 RLS 策略是否正确配置

### Q4: AI 解析失败？

**A**: 检查以下几点：
1. DEEPSEEK_API_KEY 是否正确
2. API 额度是否用完
3. 网络连接是否正常
4. 查看 GitHub Actions 日志中的详细错误

### Q5: 如何清空所有任务重新开始？

**A**: 在 Supabase SQL Editor 中执行：
```sql
-- 清空所有任务（保留游戏化数据）
DELETE FROM tasks WHERE user_email = 'your@163.com';
```

### Q6: 如何修改邮件发送时间？

**A**: 编辑 `.github/workflows/` 下的对应文件，修改 cron 表达式：
```yaml
schedule:
  - cron: '30 13 * * *'  # UTC时间，对应北京时间21:30
```

### Q7: 如何备份数据？

**A**: Supabase 提供自动备份功能：
1. 进入项目设置 → Database → Backups
2. 可以手动创建备份或下载现有备份
